# Vendor Pipeline

## What to build

Add a generic Vendor workflow with required Vendor Category and Vendor Status. A Planner should be able to add vendors such as venues, caterers, decorators, photographers, priests, planners, music, beauty, or transport providers; connect Vendors to Events and Tasks; and see a Vendor pipeline summary on the Dashboard.

## Acceptance criteria

- [ ] A Planner or Owner can create a Vendor with a required Vendor Category.
- [ ] Vendor Status supports Researching, Contacted, Quote Received, Shortlisted, Booked, Rejected, and Cancelled.
- [ ] Vendors can be assigned to one or more Events.
- [ ] Tasks can be linked to Vendors for booking and follow-up work.
- [ ] The Dashboard shows a Vendor pipeline grouped by status and category.
- [ ] Tests cover Vendor creation, required category validation, status updates, Event assignment, and pipeline summary.

## Blocked by

- Event and Task Planning Loop
