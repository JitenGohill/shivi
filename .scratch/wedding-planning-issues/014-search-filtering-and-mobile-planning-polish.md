# Search, Filtering, and Mobile Planning Polish

## What to build

Add cross-record search, filtering, and mobile workflow polish. Collaborators should be able to find planning records by owner, status, Event, Vendor Category, date, Attention Needed, and common text fields. Dashboard summaries should drill down to filtered source records. Mobile layouts should support quick planning updates during errands and family discussions.

## Acceptance criteria

- [ ] Collaborators can search across major planning records.
- [ ] Collaborators can filter records by owner, status, Event, Vendor Category, date, and Attention Needed where applicable.
- [ ] Dashboard summary cards drill down to the relevant filtered records.
- [ ] Search and filters respect Collaborator permissions.
- [ ] Core planning workflows are usable on mobile viewports.
- [ ] Tests cover search results, filter behavior, dashboard drill-downs, permissions, and representative mobile rendering.

## Blocked by

- Vendor Pipeline
- Budget and Payment Tracker
- Guest Groups, Invitations, and Event-Level RSVPs
- Documents and Notes Across Planning Records
- Planning Areas
- Timeline, Reminders, and Attention Needed
- Accommodation and Transport Planning
