# Event and Task Planning Loop

## What to build

Create the first complete planning loop around Events and Tasks. A Collaborator should be able to create Events, assign Leads, add Tasks, assign Task owners, set due dates, update Task Status, mark blockers, and see task-driven Progress roll up to Events and the Wedding Dashboard.

## Acceptance criteria

- [ ] A Planner or Owner can create and edit Events with date, location, Lead, and Notes.
- [ ] A Planner or Owner can create Tasks linked to the Wedding or a specific Event.
- [ ] A Task has owner, due date, status, and optional blocked reason.
- [ ] Contributors can update assigned Task statuses without changing Wedding structure.
- [ ] Event Progress and Wedding Progress are calculated primarily from Task Status.
- [ ] The Dashboard shows upcoming Tasks, overdue Tasks, blocked Tasks, current-user assignments, and Event Progress.
- [ ] Tests cover Task Status transitions, Progress rollups, blocked work, overdue work, and permission behavior.

## Blocked by

- Scaffold Single Wedding App Shell
- Invited Collaborator Access and Roles
